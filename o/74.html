30. Построение выпуклой оболочки многоугольника

Рассмотрим n точек на плоскости. Их выпуклой оболочкой назовем минимальное выпуклое множество, содержащее целиком эти точки. Можно представить себе выпуклую оболочку как фигуру, которая получится, если обхватить все данные точки лассо, а затем «стянуть» его. Для n точек выпуклой оболочкой будет многоугольник (очевидно, выпуклый), вершинами которого являются некоторые из данных точек.

Несложно придумать наивный алгоритм построения выпуклой оболочки: переберем все пары точек, для каждой пары точек проверим, что все остальные точки лежат по одну сторону от выбранной, если это свойство выполняется — то отрезок, соединяющий выбранные точки является одной из сторон выпуклой оболочки. Такой алгоритм будет иметь сложность O(n3).

Один из более эффективных алгоритмов построения выпуклой оболочки называется алгоритмом Грэхема. Рассмотрим этот алгоритм.

Выберем среди всех данных точек самую нижнюю (то есть точку с минимальной y-координатой), а если таких несколько — то самую левую из них. Эта точка обязательно войдет в выпуклую оболочку. Обозначим эту точку, как S. Все остальные точки упорядочим по возрастанию полярного угла, относительно точки S (то есть рассмотрим радиус-вектор SPi¯ и упорядочим точки Pi по возрастанию полярного угла данного вектора). На этом этапе удобней точку S перенести в начало координат, то есть из координат всех точек вычесть координаты точки S. Тогда все точки сортируются по возрастанию полярного угла, который будет принимать значения на интервале [0,π). Если несколько точек имеют одинаковый полярный угол, то они сортируются по увеличению расстояния до точки S. То есть для того, чтобы сортировка работала правильно, удобно каждую точку представить в виде кортежа из четырех чисел: φ, r, x, y, где φ - полярный угол точки относительно S, r - расстояние до S, x и y - координаты точки и далее сортировать указанные кортежи.

После сортировки точки рассматриваются по одной и хранится выпуклая оболочка уже рассмотренных точек в отдельном списке Q (который по сути является стеком, т. к. всегда используются только два последних элемента выпуклой оболочки). Точки выпуклой оболочки храняться в Q в положительном направлении обхода. В самом начале в список Q кладутся две первый вершины из множества P после сортировки, то есть первой вершиной в Q будет вершина S. Остальные вершины по очередит добавляются в Q, а также могут быть и удалены из Q если обнаружится, что они не должны входить в выпуклую оболочку.

Все вершины, которые находятся в Q, должны быть выпуклым многоугольником, то есть в каждой вершине из Q поворот должен осуществляться в положительном направлении. При добавлении в Q новой вершины из множества P (будем считать, что это вершина Pi), проверим, верно ли, что в последней вершине из списка Q после добавления вершины Pi в конец Q будет положительный поворот. Если последняя вершина в Q это Q−1, а предпоследняя - Q−1 (используем отрицательные индексы, как в списках), то для добавления вершины Pi необходимо, чтобы [Q−2Q−1¯,Q−1Pi¯]>0. Если это условие выполняется, то вершина Pi добавляется в конец списка P (и становится последней). Если не выполняется — то из Q нужно удалить последнюю вершину и повторить проверку. То есть при добавлении новой вершины Pi в конец Q из списка Q нужно удалять последнюю вершину, пока выполняется свойство [Q−2Q−1¯,Q−1Pi¯]⩽0, или пока в списке Q не останется только одна вершина. После чего вершина Pi добавляется в конец Q.

Нетрудно понять, что сложность такого алгоритма O(nlog⁡n), то есть совпадает со сложностью сортировки списка точек, так как оставшаяся часть алгоритма после сортировки выполняется за O(n): все точки просматриваются по одному разу, и каждая точка будет один раз добавлена в Q, и может быть не более одного раза удалена из Q. При добавлении новой точки Pi в конец Q производится удаление точек, ранее добавленных в Q, поэтому суммарная сложность обработки всех точек не более, чем O(n).
